# Map of Hubitat capability -> required command/method names.
#
# Format: CapabilityName : method1, method2, method3
# Lines starting with # are comments. Capabilities that are pure marker /
# attribute-only types (sensors) appear with an empty method list, which means
# the linter doesn't require any methods for them.
#
# Source: Hubitat Driver Capabilities documentation. Update as Hubitat ships
# new capabilities. The CapabilityRequiredMethodsRule checks that each declared
# capability's required methods exist either locally OR via an #include'd
# library.
